# PRD Step 3 — 사용자 프로필 및 레시피 저장

## 개요

사용자 프로필을 생성하고, 2단계에서 생성된 레시피를 저장·조회·삭제하는 개인화 기능을 제공한다.

---

## 목표

- 사용자별 식단 선호도와 알레르기 정보를 저장하여 레시피 추천 품질을 향상한다.
- 마음에 드는 레시피를 북마크하고 언제든지 다시 볼 수 있게 한다.
- 향후 1~2단계에 프로필 정보를 자동으로 반영하는 기반을 구축한다.

---

## 사용자 스토리

| # | As a… | I want to… | So that… |
|---|-------|-----------|----------|
| 1 | 신규 사용자 | 프로필을 등록하고 싶다 | 나의 식단 조건에 맞는 레시피를 추천받을 수 있다 |
| 2 | 사용자 | 레시피를 저장하고 싶다 | 나중에 다시 확인할 수 있다 |
| 3 | 사용자 | 저장된 레시피 목록을 보고 싶다 | 어떤 레시피를 북마크했는지 한눈에 볼 수 있다 |
| 4 | 사용자 | 저장된 레시피를 삭제하고 싶다 | 불필요한 레시피를 정리할 수 있다 |
| 5 | 사용자 | 프로필을 수정하고 싶다 | 변경된 식단 정보를 반영할 수 있다 |

## 기능 요구사항

### F1. 사용자 프로필

**프로필 정보**
| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | UUID v4 자동 생성 |
| `name` | string | 사용자 이름 (필수) |
| `dietary_preferences` | string[] | 식단 선호 (채식, 비건, 할랄 등) |
| `allergies` | string[] | 알레르기 재료 목록 |
| `disliked_ingredients` | string[] | 싫어하는 재료 목록 |
| `preferred_cook_time` | number | 선호 조리 시간 (분) |
| `servings` | number | 기본 인분 수 |
| `created_at` | ISO8601 | 생성 일시 |
| `updated_at` | ISO8601 | 수정 일시 |

**프로필 생성 화면 필드**
- 이름 (텍스트 입력, 필수)
- 식단 선호 (체크박스: 채식 / 비건 / 글루텐프리 / 유제품프리 / 할랄)
- 알레르기 (태그 입력: 예) 새우, 땅콩, 계란)
- 싫어하는 재료 (태그 입력)
- 기본 조리 시간 (드롭다운)
- 기본 인분 수 (숫자 입력)

### F2. 레시피 저장

**저장 데이터 구조**
```json
{
  "id": "saved_001",
  "user_id": "user_uuid",
  "recipe": {
    "name": "달걀 채소볶음",
    "description": "...",
    "cook_time_minutes": 20,
    "servings": 2,
    "difficulty": "쉬움",
    "ingredients": [...],
    "steps": [...],
    "tips": "...",
    "nutrition": {...}
  },
  "source_ingredients": ["달걀", "당근", "양파"],
  "tags": ["빠른요리", "채소"],
  "memo": "다음엔 간장 줄이기",
  "saved_at": "2024-01-15T12:00:00Z"
}
```

**저장 동작**
- 2단계 레시피 상세 화면의 "저장하기" 버튼 클릭
- 중복 저장 방지: 동일 레시피명이 이미 저장된 경우 알림
- 사용자 메모 추가 가능

### F3. 저장된 레시피 목록

- 그리드 또는 리스트 뷰 전환
- 정렬: 저장일 최신순 / 조리 시간 짧은순 / 이름 가나다순
- 검색: 레시피명 또는 재료명으로 필터
- 태그 필터

### F4. 레시피 삭제

- 목록에서 삭제 버튼 클릭 → 확인 다이얼로그
- 일괄 선택 삭제 지원

## 비기능 요구사항

| 항목 | 요건 |
|------|------|
| 저장소 | 로컬: `localStorage` / 서버: JSON 파일 또는 SQLite |
| 최대 저장 레시피 수 | 로컬 100개 (초과 시 오래된 순 삭제 안내) |
| 데이터 보안 | 알레르기 정보는 서버 전송 전 암호화 권고 |
| 오프라인 지원 | localStorage 기반으로 네트워크 없이도 저장 목록 조회 가능 |

## 기술 스택

| 영역 | 기술 |
|------|------|
| 저장소 (1차) | `localStorage` (브라우저 내 JSON) |
| 저장소 (2차, 선택) | Node.js + SQLite (better-sqlite3) 또는 JSON 파일 |
| 사용자 식별 | UUID v4 (crypto.randomUUID) — 별도 인증 없음 |
| 상태 공유 | 세션 스토리지로 1~2단계 프로필 자동 적용 |

## API 엔드포인트 (서버 저장소 사용 시)

### `POST /api/profile` — 프로필 생성/수정
```json
{
  "name": "홍길동",
  "dietary_preferences": ["채식"],
  "allergies": ["새우", "땅콩"],
  "preferred_cook_time": 30,
  "servings": 2
}
```

### `GET /api/profile/:userId` — 프로필 조회

### `POST /api/saved-recipes` — 레시피 저장
```json
{
  "user_id": "uuid",
  "recipe": { ... },
  "memo": "다음엔 간장 줄이기"
}
```

### `GET /api/saved-recipes/:userId` — 저장 목록 조회

### `DELETE /api/saved-recipes/:recipeId` — 레시피 삭제

## 데이터 흐름

```
[사용자 프로필 설정]
  dietary, allergies, preferences
        │
        ▼
[localStorage / 서버 저장]
        │
        ├──→ [Step 1로 전달] → 알레르기 재료 필터링
        └──→ [Step 2로 전달] → 식단 조건 자동 적용
                                        │
                                        ▼
                              [레시피 생성 결과]
                                        │
                                  [저장하기 클릭]
                                        │
                                        ▼
                              [저장된 레시피 목록]
```

---

## 1~3단계 통합 흐름

```
[Step 1] 냉장고 사진 업로드
         → 재료 인식 (Gemma)
         → 재료 목록 편집
              │
              ▼
[Step 2] 옵션 선택 + 레시피 생성 (GPT-oss-120b)
         → 레시피 3개 표시
         → 상세 보기
              │
              ▼ [저장하기]
[Step 3] 레시피 저장 (localStorage)
         → 저장 완료 메시지
         → 저장 목록에서 확인
```
